# EliGiftManager

Go + Wails desktop application scaffolded with a standard Vue 3 SFC frontend powered by Vite and Deno.

## Versions

- Go: `1.26.2`
- Wails: `v2.12.0`
- Vue: `3.5.33`
- Vite: `8.0.9`
- Deno: `2.7.12`

As of `2026-04-23`, `Wails v2.12.0` is the latest stable Wails release, `Go 1.26.2` is the latest stable Go release, and the Vue/Vite toolchain is pinned to `Vue 3.5.33` plus `Vite 8.0.9`.

## Stack

- Backend: Go + Wails
- Frontend: Vue 3 SFC + Vite + TypeScript
- Package manager and task runner: Deno
- Desktop shell: Wails asset pipeline and native window lifecycle

## Workflow

- `deno install` resolves npm dependencies and creates the local compatibility `node_modules` directory required by the Vite toolchain.
- `wails dev` uses the `frontend:dev:watcher` command declared in `wails.json` and connects to `http://127.0.0.1:5173`.
- `frontend/wailsjs` is generated by Wails and should be treated as build-managed code, not handwritten application logic.
- `frontend/dist` is a production artifact generated by Vite and is intentionally ignored.

## Quick Start

```powershell
# install backend dependencies
go mod tidy

# install frontend dependencies through Deno
Set-Location frontend
deno install

# run type checking or build the frontend
deno task typecheck
deno task build

# return to the project root and start Wails development
Set-Location ..
wails dev

# or build the packaged desktop binary
wails build
```

## Structure

- `internal/config`: backend application metadata and window settings
- `frontend/src/app`: Vue application shell and router
- `frontend/src/pages`: route-level screens
- `frontend/src/features`: feature-oriented components and model constants
- `frontend/src/shared`: shared UI, types, and Wails integration wrappers
- `frontend/package.json`: Vite and Vue toolchain metadata
- `frontend/deno.json`: Deno task runner and npm compatibility settings
- `build`: Wails packaging assets
- `docs`: repository conventions and structure notes

## Generated Files

- `frontend/node_modules`: generated by `deno install`, ignored
- `frontend/package.json.md5`: generated by Deno npm compatibility bookkeeping, ignored
- `frontend/wailsjs`: generated by Wails bindings, committed because the frontend imports it
- `build/bin`: generated packaged binaries, ignored
