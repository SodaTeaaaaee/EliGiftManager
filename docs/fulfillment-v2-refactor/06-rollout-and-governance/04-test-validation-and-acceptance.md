# 测试、验证与验收

本文件整理必测业务场景、回归重点与验收标准，作为后续重构落地的质量门槛。

## 13. 测试与验证计划

### 13.1 必测业务场景

1. 纯会员波次
2. 纯零售订单波次
3. 混合波次
4. 多工厂平台同波次
5. 工厂部分发货
6. 多包裹
7. 物流回填成功
8. 物流回填失败后重试
9. 会员无订单号场景
10. 买家无会员身份场景
11. `policy_driven` 规则推导场景
12. `demand_driven` 订单直入场景
13. mixed wave 中同时存在 `policy_driven` 与 `demand_driven` 路径
14. 平台权威已判定成立的阶段性会员礼物
15. 会员地址晚于权益成立才补齐
16. 会员通过表单或协商补充款式/尺码/颜色
17. 会员限定购买作为 `retail_order` 进入系统
18. `excluded_manual` 被单独统计但不进入执行链
19. `deferred` 需求后续重新纳入本系统
20. 手工录入的会员权益仍以 `membership_entitlement + manual_entry` 进入系统
21. 手工录入的线下订单仍以 `retail_order + manual_entry` 进入系统
22. 混合波次经 `Wave Overview` 汇总后再进入共享调整层
23. 从 `Wave Overview` 直接跳过共享调整层进入后续阶段
24. 在 `Membership Allocation` / `Demand Mapping` / `Adjustment Review` 之间来回跳转而不破坏已有结果
25. 前置页面重算后，已确认的共享调整层例外仍被正确重放
26. 试图在共享调整层直接新增全新参与者时，被系统明确阻止或引导回前置层
27. 工厂导出后继续修改内容时，系统给出“导出快照已过期”类提示，但不锁死编辑
28. 导入物流回传后继续修改内容时，系统给出“当前结果已偏离最近一次物流映射基础”类提示，但不锁死编辑
29. 前置层重算导致已确认的共享调整层例外失配时，系统会提示复核，而不是静默吞掉例外
30. `Wave Overview` 能聚合展示偏离与复核信号，但不把这些信号当成不可编辑锁
31. 会员规则贡献里出现负数，或最终结果被压到 0 时，系统不会仅因出现负号就误报 review
32. 试图在 `Adjustment Review` 里用身份 / 平台 / 交集 selector 做动态批量修正时，被明确阻止并引导回 `Membership Allocation`
33. 从某个旧历史节点分叉重新编辑后，旧未来不会丢失，而是保留为另一条分支
34. 关闭软件再打开后，`wave` 工作区历史仍可恢复
35. undo / redo 不会伪装成已真实撤销工厂导出、物流导入或渠道同步
36. 在文本输入框中使用 `Ctrl+Z / Ctrl+Shift+Z` 时，优先触发输入框自身的原生撤销 / 重做
37. undo / redo 后，用户能收到即时 toast，并在短时间内翻看最近动作回执

### 13.2 回归重点

重点回归以下老能力：

- 地址绑定
- 波次重算
- 商品快照与主档关系
- 用户补发/减发覆盖
- 工厂导出模板
- 当前会员规则分配与批量编辑体验
- 会员输入采集与地址晚补场景
- 未纳入本系统处理项的单独统计
- 跨步骤往返导航下的数据稳定性
- 前置重算与调整层例外并存时的结果稳定性
- 导出/回传对象与当前工作区结果偏离时的提示稳定性
- `wave` 工作区树状历史的分支保留稳定性
- 历史持久化恢复后的 head / basis 对齐稳定性
- undo / redo 反馈层（toast / 回执）的一致性

### 13.3 验收标准

V2 至少应满足：

1. 能同时承接会员和买家两类来源
2. 能把履约来源追踪到具体需求单和需求行
3. 能持久化记录工厂提交结果
4. 能持久化记录物流回传
5. 能把物流回填状态作为独立流程追踪
6. 波次状态能真实反映完整生命周期
7. 模板系统能为新增平台提供明确扩展入口
8. 会员权益型分配体验不明显劣化于当前版本
9. 支持者限定购买不会被误判成 `membership_entitlement`
10. 平台权威已判定的阶段性会员礼物不会被本系统错误本地重算
11. 手工录入上游事实不会被误建模成波次内调整层
12. `excluded_manual` 等未纳入项会进入统计，但不会伪装成已执行或待发货状态
13. `Wave Overview` 能独立承载波次内阶段导航、异常聚合与下一步判断，而不与复杂编辑层混杂
14. 波次跨步骤跳转不会导致隐式的数据破坏或不可解释的重算结果
15. 共享调整层确认过的例外，在前置层重算后仍具有可预期的保留规则
16. 共享调整层不能退化为手工终表编辑器
17. 导出或回传之后的继续编辑仍被允许，但系统能提示当前结果已偏离最近一次辅助快照
18. 物流回传或渠道回填之后再次修改结果时，系统仍只发出辅助提示而不形成硬锁
19. 前置层重算导致例外失配时，系统会进入复核语义，而不是悄悄吞掉或自动篡改既有调整
20. 动态 selector 规则仍停留在 `Membership Allocation`，共享调整层不会重新长成第二套规则引擎
21. 页面只是 UX 入口，领域语义真正归属稳定的 service / projection 层
22. 树状 undo / redo 在跨重启后仍保留分支与当前 head
23. 工作区级 undo / redo 与外部 basis 偏离提示可以协同存在，而不会相互伪装
24. 用户在执行 undo / redo 后，总能得到明确、短期可回看的操作反馈
